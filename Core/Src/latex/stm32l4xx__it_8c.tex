Führt den Context Switch (Kontextwechsel) durch.

Diese Funktion ist der Pend\+SV-\/\+Interrupt-\/\+Handler. Das Attribut {\ttfamily \doxylink{stm32l4xx__it_8c_a2804a023941a956288c32ad08b2cf59e}{\+\_\+\+\_\+attribute\+\_\+\+\_\+((naked))}} verhindert, dass der Compiler eigenen Prolog/\+\+Epilog-\/\+Code generiert. Die Funktion sichert die verbleibenden Software-\/\+Register (r4-\/r11) des aktuell laufenden Tasks auf dessen Stack, speichert den aktuellen Stackpointer (MSP) im Task Control Block (TCB) und ruft den {\ttfamily \doxylink{stm32l4xx__it_8c_a8476cf7898494fd1499c79769d65ed34}{OS\+\_\+\+Scheduler}} auf, um den nächsten Task zu bestimmen. Danach lädt sie den Stackpointer des neuen Tasks, stellt dessen Software-\/\+Register wieder her und triggert über den Rücksprung ({\ttfamily bx lr} mit EXC\+\_\+\+RETURN) die automatische Wiederherstellung der Hardware-\/\+Register durch die CPU. Behandelt den System Tick Timer Interrupt (Sys\+Tick). 

Diese Funktion wird periodisch von der Hardware aufgerufen (z.\+B. jede Millisekunde). Sie übernimmt neben der grundlegenden HAL-\/\+Zeitbasis ({\ttfamily HAL\+\_\+\+Inc\+Tick}) drei zentrale Aufgaben für das RTOS:\+
\begin{DoxyEnumerate}
\item Sie verringert die Timeout-\/\+Zähler ({\ttfamily u32\+Timeout\+Ticks}) von blockierten Tasks (z. B. Wartezeiten bei Semaphoren/\+\+Queues) und weckt diese bei Ablauf auf.
\item Sie aktualisiert die Delay-\/\+Zähler ({\ttfamily u32\+Delay\+Ticks}) von pausierten Tasks (ausgelöst durch {\ttfamily \doxylink{os__kernel_8c_ab5ed9279d3faff181a277594e53ac5af}{OS\+\_\+\+Delay}}) und setzt diese bei Ablauf wieder auf {\ttfamily Task\+State\+\_\+\+Ready}.
\item Sie verwaltet das Time-\/\+Slicing für das präemptive Round-\/\+Robin-\/\+Scheduling. Sobald die festgelegte Zeitscheibe ({\ttfamily g\+\_\+u8\+Praemtive\+Time}) abgelaufen ist, wird ein Pend\+SV-\/\+Interrupt ausgelöst, um den Context Switch zu erzwingen.
