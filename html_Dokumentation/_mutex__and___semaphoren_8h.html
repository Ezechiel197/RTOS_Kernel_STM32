◆ OS_Mutex_AcquireBlocking()

void OS_Mutex_AcquireBlocking ( OS_Mutex_t * pMutex)

Fordert einen Mutex an (Blocking).

Ist der Mutex belegt, wird der aktuelle Task in den Zustand TaskState_Blocked versetzt und in der Wartemaske des Mutex registriert. Anschließend wird ein Context Switch (PendSV) erzwungen.

Parameters
pMutexZeiger auf das angeforderte Mutex-Objekt.

◆ OS_Mutex_AcquireNonBlocking()

uint8_t OS_Mutex_AcquireNonBlocking ( OS_Mutex_t * pMutex)

Fordert einen Mutex an (Non-Blocking).

Versucht, den Mutex zu sperren. Ist dieser bereits belegt, kehrt die Funktion sofort zurück, ohne den aufrufenden Task zu blockieren.

Parameters
pMutexZeiger auf das angeforderte Mutex-Objekt.
Returns
1 bei erfolgreichem Acquire, 0 wenn der Mutex bereits belegt ist.

◆ OS_Mutex_AcquireTimeout()

uint8_t OS_Mutex_AcquireTimeout ( OS_Mutex_t * pMutex,
uint32_t u32TimeoutTicks )

Fordert einen Mutex mit Timeout an.

Blockiert den Task maximal für die angegebene Zeitspanne (Ticks). Wird der Task durch einen anderen Task geweckt (Release), erhält er den Mutex. Läuft der Timer ab (SysTick), schlägt das Acquire fehl.

Parameters
pMutexZeiger auf das angeforderte Mutex-Objekt.
u32TimeoutTicksMaximale Wartezeit in OS-Ticks.
Returns
1 bei erfolgreichem Acquire, 0 bei einem Timeout.

◆ OS_Mutex_Init()

void OS_Mutex_Init ( OS_Mutex_t * pMutex)

Initialisiert ein Mutex-Objekt.

Setzt den Lock-Status auf frei und weist eine ungültige Owner-ID (255) zu, um zu signalisieren, dass der Mutex aktuell keinem Task gehört.

Parameters
pMutexZeiger auf das zu initialisierende OS_Mutex_t Objekt.

◆ OS_Mutex_Release()

void OS_Mutex_Release ( OS_Mutex_t * pMutex)

Gibt einen Mutex frei.

Überprüft zunächst, ob der aufrufende Task tatsächlich der Besitzer ist. Sind andere Tasks in der Warteliste, wird der Mutex direkt an den wartenden Task mit der entsprechenden Priorität (bzw. ID) übergeben und ein Context Switch angefordert.

Parameters
pMutexZeiger auf das freizugebende Mutex-Objekt.

◆ OS_Sem_AcquireTimeout()

uint8_t OS_Sem_AcquireTimeout ( OS_Semaphore_t * pSem,
uint32_t u32TimeoutTicks )

Fordert ein Semaphore-Token an mit Timeout.

Versetzt den Task in den Blocked-Status, falls kein Token vorhanden ist. Der Task wacht entweder durch ein Timeout (ausgelöst vom SysTick) oder durch die Freigabe eines Tokens (Release) wieder auf.

Parameters
pSemZeiger auf das angeforderte Semaphore-Objekt.
u32TimeoutTicksMaximale Wartezeit in OS-Ticks.
Returns
1 wenn das Token innerhalb der Zeit erhalten wurde, 0 bei Timeout.

◆ OS_Sem_Release()

void OS_Sem_Release ( OS_Semaphore_t * pSem)

Gibt ein Semaphore-Token frei oder signalisiert ein Event.

Wenn sich Tasks in der Warteschlange befinden, wird der Zähler (Count) nicht erhöht. Stattdessen wird direkt ein wartender Task geweckt (TaskState_Ready) und ein Context Switch angefordert. Andernfalls wird der Token-Zähler inkrementiert.

Parameters
pSemZeiger auf das freizugebende Semaphore-Objekt.
